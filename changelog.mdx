## Recent Updates

## Upgrading to the Latest Version

Follow these steps to upgrade Self-comp seamlessly.

<Callout kind="alert">
Breaking changes in v1.1.0 require updating API calls from `/v1/` to `/v2/`. Review your integrations.
</Callout>

<Steps>
  <Step title="Backup Your Data" icon="database">
    Export all documents via the dashboard at `https://dashboard.example.com/export`.
  </Step>
  <Step title="Update SDK" icon="download">
    Install the latest version using your package manager.
    
    <CodeGroup tabs="npm,yarn">
    ````bash
    npm install @selfcomp/sdk@latest
    ````
    ````bash
    yarn add @selfcomp/sdk@latest
    ````
    </CodeGroup>
  </Step>
  <Step title="Migrate API Calls" icon="settings">
    Update endpoints in your code.
    
    <Tabs>
      <Tab title="Before (v1)" icon="x-circle">
      ````javascript
      const response = await fetch('https://api.example.com/v1/documents');
      ````
      </Tab>
      <Tab title="After (v2)" icon="check-circle">
      ````javascript
      const response = await fetch('https://api.example.com/v2/documents');
      ````
      </Tab>
    </Tabs>
  </Step>
  <Step title="Test and Deploy" icon="rocket">
    Run tests and redeploy your application.
  </Step>
</Steps>

## Backward Compatibility

Self-comp maintains backward compatibility for minor releases (e.g., v1.2.x to v1.3.x). Major versions may introduce breaking changes, noted in the `<Update>` tags with `"breaking"`. You always access deprecation warnings in API responses.

<ParamField path="api-version" param-type="string" required="true">
  Specify the version header: `X-API-Version: 2.0`.
</ParamField>
