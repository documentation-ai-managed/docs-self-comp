## Prerequisites

Before you begin, ensure you have the following:

<Callout kind="info">

- A modern web browser (Chrome, Firefox, Safari, or Edge)
- An email address for account verification
- Optional: Existing documentation files in Markdown or GitHub repository

</Callout>

## Create Your Account

Follow these steps to sign up and access the Self-comp dashboard.

<Steps>
  <Step title="Visit Dashboard" icon="globe">
    Navigate to `https://dashboard.example.com` and click **Sign Up**.
  </Step>
  <Step title="Enter Details" icon="user">
    Provide your email, create a strong password, and complete the CAPTCHA.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for the verification link from Self-comp and click it to activate your account.
  </Step>
  <Step title="Log In" icon="log-in">
    Return to the dashboard, enter your credentials, and land on the home screen.
  </Step>
</Steps>

## Set Up Your First Space

Spaces organize your documentation projects. Create one now.

<Steps>
  <Step title="Create Space" icon="plus">
    From the dashboard, click **New Space**. Name it `My Project Docs` and select a description.
  </Step>
  <Step title="Configure Settings" icon="settings">
    Choose visibility (`public` or `private`) and add team members if needed.
  </Step>
  <Step title="Save and Enter" icon="check-circle">
    Click **Create**. You now enter your space's editor interface.
  </Step>
</Steps>

## Import Initial Documentation

Bring in existing content quickly using these methods.

<Tabs>
  <Tab title="Manual Upload" icon="upload">
    Drag and drop Markdown files into the editor.

    <CodeGroup tabs="Markdown">
      ````markdown
      # Welcome to My Docs

      This is your first page.

      ## Features

      - Real-time collaboration
      - MDX component support
      ````
    </CodeGroup>
  </Tab>
  <Tab title="GitHub Repo" icon="github">
    Connect your repository via the import wizard.

    1. Click **Import from GitHub**.
    2. Authorize Self-comp.
    3. Select repo and branch (e.g., `main`).

    Self-comp syncs changes automatically.
  </Tab>
  <Tab title="API Import" icon="code">
    Use the API for programmatic imports.

    <Request tabs="cURL,JavaScript">
      ````bash
      curl -X POST https://api.example.com/v1/spaces/{SPACE_ID}/import \
        -H "Authorization: Bearer YOUR_TOKEN" \
        -H "Content-Type: application/json" \
        -d '{"url": "https://raw.githubusercontent.com/user/repo/main/README.md"}'
      ````

      ````javascript
      const response = await fetch(`https://api.example.com/v1/spaces/${SPACE_ID}/import`, {
        method: 'POST',
        headers: {
          'Authorization': 'Bearer YOUR_TOKEN',
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ url: 'https://raw.githubusercontent.com/user/repo/main/README.md' })
      });
      ````
    </Request>
  </Tab>
</Tabs>
